使用郵件應用程式

Note

郵件應用程式預設與 Nextcloud Hub 一起安裝，但可以禁用。請向您的管理員詢問。

管理您的郵件帳戶

切換佈局

Added in version 3.6.

  1. 訪問郵件設置

  2. 選擇 列表、垂直分割 或 水平分割

使用精簡模式

Added in version 5.7.0.

精簡模式提供更簡潔及高效的訊息檢視方式。頭像會被隱藏、選取方塊會一直顯示，並移除訊息預覽。此模式可節省空間，讓你同時查看更多電郵。

  1. 前往郵件設定

  2. 前往 Appearance

  3. 切換精簡模式

訊息顯示 / 操作模式

Added in version 5.2.

郵件可以在兩種不同的消息查看和操作模式之間切換：主題式 和 單一式。

在 主題式 模式下，消息按對話分組。在郵箱消息列表中，相關的消息會堆疊，因此只顯示最新的消息，並且在選擇堆疊的消息後，所有相關的消息會顯示在消息顯示面板中。這對於跟進討論和理解回覆的上下文非常有用。在此模式下，移動和刪除等消息操作適用於整個主題，這意味著當你移動或刪除主題時，該主題內的所有消息都會受到影響。

在 單一式 模式下，消息會單獨顯示，在郵箱消息列表和消息顯示面板中，移動和刪除等操作僅適用於所選的消息。當你希望單獨管理消息而不影響整個對話時，這種模式非常有用。

  1. 訪問郵件設置

  2. 選擇 主題式 或 單一式

添加新郵件帳戶

  1. 從應用中啟用郵件應用程式

  2. 點擊標頭上的郵件圖標

  3. 填寫登錄表單（自動或手動）

更改排序順序

Added in version 3.5.

  1. 訪問郵件設置

  2. 轉到 排序

  3. 您可以選擇優先顯示 最舊 或 最新 郵件

Note

此更改將適用於您所有的帳戶和郵箱

將最愛置頂

Added in version 5.7: Nextcloud 31 或以上版本

此設定可將標記為最愛的訊息顯示於訊息列表頂部的獨立區域。

  1. 訪問郵件設置

  2. 前往 Appearance

  3. 啟用最愛置頂

預定消息

  1. 點擊屏幕左上角的新消息按鈕

  2. 點擊模態編輯器上的（...）操作選項單

  3. 點擊 稍後發送

優先收件箱

優先收件箱有 2 個部分，重要 和 其他。根據您互動或標記為重要的消息，消息將自動標記為重要。一開始，您可能需要手動更改重要性以教導系統，但隨著時間的推移，它將會改善。

自動分類為選用功能。你可於設定帳戶時選擇停用，亦可隨時於帳戶設定中開啟或關閉。

所有收件箱

您登錄的所有帳戶的所有消息將按時間順序顯示在這裡。

帳戶設置

您的帳戶設置包括：

  1. 別名

  2. 簽名

  3. 默認資料夾

  4. 自動回覆

  5. 受信任的發件人

  6. ..以及更多

可以在郵件帳戶的操作選項單中找到。在那裡，您可以根據需要編輯、添加或刪除設置。

將消息移動到垃圾郵件資料夾

Added in version 3.4.

郵件可以在消息被標記為垃圾時將其移動到不同的資料夾。

  1. 存取帳戶設置

  2. 轉到默認資料夾

  3. 檢查是否已選擇一個資料夾用於垃圾郵件

  4. 轉到垃圾郵件設置

  5. 點擊將消息移動到垃圾郵件資料夾

重新整理信箱

你可以點擊信箱列表頂部的重新整理按鈕，手動觸發同步。自 version 5.7 起，執行同步亦會更新所選帳戶的資料夾列表。

統一搜尋

Mail 應用程式已整合 Nextcloud 的 統一搜尋 功能（詳情請參閱 Nextcloud Web 介面）。你可使用 Nextcloud 標頭的搜尋列，在所有帳戶中搜尋電郵。搜尋

Mail 會搜尋電郵主旨及寄件人／收件人欄位。如需搜尋電郵內容，請使用應用程式內的信箱搜尋功能。

在郵箱中搜索

Added in version 2.1.

在任何郵件佈局的信封列表頂部，有一個搜索字段快捷鍵，用於搜索郵件主題。從 版本 3.7 開始，此快捷鍵默認允許您按主題、收件人（發送到）或發件人（來自）進行搜索。

在郵箱中進行高級搜索

Added in version 3.4.

您可以通過位於搜索快捷鍵末尾的模態訪問我們的高級搜索功能。

啟用郵件內容搜索

Added in version 3.5.

現在可以搜索郵件內容，此功能是選擇性啟用的，因為可能會有性能問題。

啟用方法：

  1. 存取帳戶設置

  2. 轉到郵箱搜索

  3. 啟用郵件內容搜索

Warning

如果您還想為統一郵箱啟用此功能，必須在郵件設置中進行。

啟用後，主搜索框將同時搜索主題和郵件內容，並且高級搜索中將出現單獨的 內容 選項。

帳戶委派

該應用程式允許帳戶委派，這樣一個用戶可以從另一個用戶的地址發送電子郵件。

  1. 委派必須由管理員在郵件伺服器上配置

  2. 將另一個電子郵件地址添加為您自己電子郵件帳戶的別名

  3. 發送電子郵件時，選擇別名作為發件人

Warning

如果發送的電子郵件存儲在您的個人 已發送 郵箱中，則原始帳戶可能無法看到該電子郵件。

自動垃圾清除

Added in version 3.4.

郵件應用程式可以在一定天數後自動刪除垃圾資料夾中的消息。

  1. 存取帳戶設置

  2. 轉到自動垃圾清除

  3. 輸入消息應刪除的天數

通過將字段留空或設置為 0 來禁用垃圾保留。

Note

只有在啟用垃圾保留後刪除的郵件將被處理。

撰寫消息

  1. 點擊您屏幕左上角的新消息

  2. 開始撰寫您的消息

最小化編輯器模態窗口

Added in version 3.2.

在撰寫新消息、編輯現有草稿或發件箱中的消息時，可以最小化撰寫窗口。只需單擊窗口右上角的最小化按鈕或單擊窗口外的任何位置。

您可以通過單擊屏幕右下角指示器的任何地方來恢復最小化的消息。

按窗口上的關閉按鈕或右下角的指示器可停止編輯消息。草稿將自動保存到你的草稿郵箱中。

撰寫器中的收件人資訊

Added in version 4.2.

當您在「收件人」字段中添加第一個收件人或聯絡人時，將出現右側面板，顯示該聯絡人的保存配置詳細信息。添加第二個聯絡人會將列表折疊，您可以選擇並展開任何您添加的聯絡人以查看其詳細信息。如果您希望專注於撰寫郵件，可以通過點擊右上角的展開圖標來隱藏右側面板。要再次顯示右側面板，只需點擊同一位置的最小化圖標。

提及聯絡人

Added in version 4.2.

您可以通過輸入 @ 來提及消息中的聯絡人，然後從列表中選擇該聯絡人。這樣，該聯絡人將自動添加為收件人。

Note

僅會建議具有有效電子郵件地址的聯絡人。

文字區塊

Added in version 5.2.

文本塊是可以插入到電子郵件中的預定義文本片段。它們可以在郵件設置中創建和管理。通過輸入 ! 然後從列表中選擇文本塊或從撰寫操作中插入文本塊。文本塊可以與用戶和用戶組共享。

發件箱

當消息已撰寫並且點擊了「發送」按鈕後，該消息會被添加到發件箱中，您可以在左側邊欄的左下角找到它。

您還可以將發送操作的日期和時間設置為未來的某個時間點（請參見 排定的消息）—該消息將保留在發件箱中，直到選定的日期和時間到達，然後自動發送。

當發件箱中有待處理的消息時，發件箱才會顯示。

在發送操作被觸發之前，您可以隨時重新打開發件箱中的消息撰寫器。

Note

當發送過程中發生錯誤時，可能會出現三條錯誤消息：

無法複製到「已發送」郵箱

郵件已發送，但無法複製到「已發送」郵箱。此錯誤將由發件箱處理，並將再次嘗試複製操作。

郵件伺服器錯誤

發送未成功，但狀態可以重試（例如：無法訪問 SMTP 伺服器）。發件箱將重試發送該消息。

消息無法發送

發送可能成功也可能失敗。郵件伺服器無法告訴我們消息的狀態。由於郵件應用無法確定消息的狀態（已發送或未發送），該消息將保留在發件箱中，帳戶用戶必須決定如何繼續。

添加郵箱

  1. 打開帳戶的操作選項單

  2. 點擊添加郵箱

添加子郵箱

  1. 打開郵箱的操作選項單

  2. 點擊添加子郵箱

如果有郵箱與您共享並具有某些特定權限，該郵箱將顯示為帶有共享圖標的新郵箱，如下所示：

信封操作

創建事件

通過郵件應用程序直接為某條消息/主題創建事件

  1. 打開信封的操作選項單

  2. 點擊 更多操作

  3. 點擊 創建事件

Note

如果管理員已啟用，將為您創建事件標題和議程。

創建任務

Added in version 3.2.

通過郵件應用程序直接為某條消息/主題創建任務

  1. 打開信封的操作選項單

  2. 點擊 更多操作

  3. 點擊 創建任務

Note

任務存儲在支持的日曆中。如果沒有兼容的日曆，您可以使用 日曆應用 創建一個新日曆。

編輯標籤

  1. 打開信封的操作選項單

  2. 點擊 編輯標籤

  3. 在標籤模態窗口中，設置/取消標籤

更改標籤顏色

Added in version 3.5.

創建標籤時，會自動選擇隨機分配的顏色。一旦標籤保存，您可以根據您的喜好自定義其顏色。此功能可以在標籤模態操作菜單中找到。

刪除標籤

Added in version 3.5.

您現在可以刪除之前創建的標籤。要訪問此功能：

  1. 打開信封/主題的操作菜單。

  2. 選擇編輯標籤。

  3. 在標籤模態中，打開您希望刪除的具體標籤的操作菜單。

Note

請注意，默認標籤如工作、待辦、個人和稍後無法刪除，僅能重命名。

AI 摘要

Added in version 4.2.

在查看郵箱時，您將看到一個短的 AI 生成的郵件摘要作為預覽。

Note

請注意，此功能必須由管理員啟用。

快速操作

Added in version 5.5: （Nextcloud 30）

可將你平常對電郵執行的操作（如標籤、移動、標記為已讀等）組合為快速操作，並可一鍵執行。快速操作僅適用於單一郵件帳戶，並可於郵件設定中的「Quick actions」或電郵操作選單中建立及管理。

Note

某些操作（如 Mark as spam、Move thread 及 Delete thread）互相排斥，不能同時加入同一快速操作，亦不可重新排序，並會最後執行。

Note

請注意，當對討論串中的其中一則訊息執行快速操作時，該操作會套用至整個討論串中的所有訊息。

消息操作

取消訂閱郵件列表

Added in version 3.1.

某些郵件列表和新聞通訊允許輕鬆取消訂閱。如果郵件應用檢測到來自此類發件人的消息，則會在發件人信息旁顯示 取消訂閱 按鈕。點擊並確認以取消訂閱。

稍後再處理

Added in version 3.4.

稍後再處理消息或線程會將其移動到專用的郵箱中，直到選定的稍後再處理日期到達，然後消息或線程將移回原始郵箱。

  1. 打開信封或線程的操作選項單

  2. 點擊 稍後再處理

  3. 選擇消息或線程應稍後再處理的時間長度

智能回覆

Added in version 3.6.

當您在郵件應用中打開一條消息時，它會提供 AI 生成的回覆。只需點擊建議的回覆，撰寫器將自動填充該回覆。

Note

請注意，此功能必須由管理員啟用。

Note

支持的語言取決於所使用的大型語言模型

郵件翻譯

Added in version 4.2.

您可以將消息翻譯成您配置的語言，方式類似於 Talk。

Note

請注意，翻譯功能必須在伺服器上啟用。

Note

自版本 5.3 起，如果管理員啟用了 LLM，將會建議翻譯。

主題摘要

郵件應用支持總結包含 3 條或更多消息的主題。

Added in version 3.4.

Note

請注意，此功能必須由管理員啟用。

Note

請注意，這一功能僅在 integration_openai 下運行良好。本地 LLM 的響應時間過長，摘要請求可能會超時並造成顯著的系統負載。

過濾器和自動回覆

郵件應用擁有 Sieve 腳本的編輯器，以及配置自動回覆和過濾器的界面。必須在 帳戶設置 中啟用 Sieve。

自動回覆

Added in version 3.5: 自動回覆可以遵循系統設置。

自動回覆默認為關閉狀態。可以手動設置，也可以遵循系統設置。遵循系統設置意味著在 缺席設置部分 中輸入的長期缺席消息將自動應用。

過濾器

Added in version 4.1.

郵件 4.1 包含一個簡單的編輯器來配置過濾器規則。

Note

不支持導入現有過濾器。但是，所有現有過濾器將保持活動且不變。我們建議通過 Sieve 腳本編輯器備份您當前的腳本，以防萬一。

如何添加新過濾器

  1. 打開您的帳戶設置。

  2. 檢查您的帳戶是否啟用了 Sieve（請參見 Sieve 伺服器設置）。

  3. 點擊過濾器。

  4. 選擇「新建過濾器」來創建新規則。

如何刪除過濾器

  1. 打開您的帳戶設置。

  2. 確保您的帳戶已啟用 Sieve（請參見 Sieve 伺服器設置）。

  3. 點擊過濾器。

  4. 將鼠標懸停在您希望刪除的過濾器上，然後點擊垃圾桶圖標。

條件

條件應用於你郵件伺服器上的來信，針對主題、發件人和收件人等字段。你可以使用以下運算符來定義這些字段的條件：

  • 完全相符：精確匹配。該字段必須與提供的值完全相同。

  • 包含：子字符串匹配。如果提供的值包含在該字段中，則該字段匹配。例如，「report」將匹配「port」。

  • 匹配：使用通配符進行模式匹配。「*」符號表示任意數量的字符（包括零），而「?」表示恰好一個字符。例如，「report」將匹配「Business report 2024」。

操作

當指定的測試為真時，將觸發操作。可用的操作如下：

  • fileinto：將消息移動到指定文件夾。

  • addflag：向消息添加標誌。

  • stop：停止過濾器腳本的執行。此操作後不會處理任何其他過濾器。

從訊息創建過濾器

Added in version 5.2.

要從特定消息創建過濾器，請打開該消息，然後點擊三個點打開菜單。接下來，點擊"更多操作"，然後選擇"創建郵件過濾器"。

在對話框中，請選擇匹配來信的條件，然後繼續點擊"創建郵件過濾器"。

跟進提醒

Added in version 4.0.

郵件應用將在發出的電子郵件未收到回覆時自動提醒您。每封已發送的電子郵件將由 AI 進行分析，以檢查是否預期會有回覆。四天後，所有相關電子郵件將顯示在您的優先收件箱中。

點擊此類電子郵件時，將顯示一個按鈕以快速跟進所有收件人。對於已發送的電子郵件，也可以禁用跟進提醒。

Note

請注意，此功能必須由管理員啟用。

安全性

釣魚檢測

Added in version 4.0.

郵件應用將檢查潛在的釣魚嘗試，並向用戶顯示警告。

檢查項目如下：

  • 地址簿中保存的發件人地址與郵件帳戶中的地址不相同

  • 發件人使用的自定義電子郵件地址與發件人地址不匹配

  • 發送日期設置在未來

  • 消息正文中的鏈接未指向顯示的文本

  • 回覆地址與發件人地址不相同

Note

請注意，該警告並不意味著消息是釣魚嘗試。它僅表示郵件應用檢測到潛在的釣魚嘗試。

內部地址

Added in version 4.0.

郵件應用允許添加內部地址和域，並在發送和接收消息時警告用戶，如果地址不在列表中。

要添加內部地址：

  1. 打開郵件設置

  2. 導航到隱私和安全部分

  3. 通過勾選復選框啟用內部地址

  4. 點擊添加內部地址按鈕

  5. 輸入地址或域名，然後單擊添加

儀表板整合

Added in version 1.8.

郵件應用提供兩個小部件，旨在與 Nextcloud 的儀表板集成：

  • 未讀郵件：此小部件顯示未讀電子郵件。

  • 重要郵件：此小部件顯示已標記為重要的電子郵件。

這些小部件利用你帳戶中設置的電子郵件帳戶中的電子郵件。

日曆整合

Mail 應用程式與日曆應用程式整合，協助你管理會議邀請並保持日曆最新。

會議邀請

當你收到包含會議邀請的訊息時，Mail 會自動偵測附加的日曆檔案，並顯示格式化操作區域以協助回應。

你可以：

  • **接受**邀請

  • **拒絕**邀請

  • **暫定接受**邀請

你的回應會直接從 Mail 應用程式發送，並相應加入至你的主要日曆。

你亦可人手將會議邀請加入至指定日曆：

  1. 開啟包含會議邀請的訊息

  2. 捲動至訊息底部的附件區域

  3. 選擇日曆檔案（通常為 .ics 格式），然後點擊三點選單

  4. 點擊「Import in to calendar」，並選擇目標日曆

會議邀請自動化

當會議主辦者更新現有活動（如時間或地點變更），Mail 會自動處理並更新你日曆中的相關事件。

Added in version 5.7: （Nextcloud 32 或以上版本）

你亦可設定 Mail 自動將所有新會議邀請加入日曆，而無需手動接受。這些邀請會以暫定狀態加入。

若要啟用此功能：

  1. 前往指定郵件帳戶的帳戶設定

  2. 前往日曆設定部分

  3. 啟用 Automatically create tentative appointments in calendar

Note

啟用後，邀請仍會顯示於郵件列表，但會自動加入日曆。

鍵盤快捷鍵

Mail 應用程式提供多個鍵盤快捷鍵，以提升操作效率。

完整快捷鍵列表可於實例中的 Mail 設定查看。
