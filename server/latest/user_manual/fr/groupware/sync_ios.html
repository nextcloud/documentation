Synchronisation avec iOS

Calendrier

  1. Ouvrir l’application Réglages.

  2. Sélectionner Applications.

  3. Sélectionner Calendrier.

  4. Sélectionner Comptes Agenda.

  5. Sélectionner Ajouter un compte.

  6. Sélectionner Autre comme Type de compte.

  7. Sélectionner Ajouter un compte CalDAV.

  8. Pour le serveur, tapez le nom de domaine de votre serveur, par ex. « exemple.com ».

  9. Saisir votre nom d’utilisateur et le mot de passe.

  10. Sélectionner Suivant.

  11. Ouvrir les Paramètres Avancés

  12. For server, type the domain name of your server and path, i.e., example.com/remote.php/dav/principals/users/username/ (replace example.com and username).

  13. Fermer les Paramètres Avancés

Votre calendrier sera désormais visible dans l’application Calendrier.

Note

Si vous avez un message d’erreur SSL, vous pouvez essayer ceci : vérifiez que vous avez indiqué et le protocole (https://) et le port (généralement 443) dans le champ Serveur, ex. https://exemple.com:443/remote.php/dav/principals/users/username/, ou aucun, comme dans le guide étape par étape ci-dessus. Autre méthode, l’application essaie automatiquement d’utiliser SSL, que vous pouvez confirmer dans les « Paramètres Avancés » du compte après avoir sauvegardé.

Note

À partir de iOS 12, un cryptage SSL est nécessaire. Par conséquent, ne désactivez pas le SSL (Pour cette raison, un certificat est requis pour votre domaine, comme https://letsencrypt.org/ ).

Contacts

  1. Ouvrir l’application Réglages.

  2. Sélectionner Applications.

  3. Sélectionner Contacts.

  4. Sélectionner Comptes Contacts.

  5. Sélectionner Ajouter un compte.

  6. Sélectionner Autre comme Type de compte.

  7. Sélectionner Ajouter un compte CardDAV.

  8. For server, type the domain name of your server and path, i.e., example.com/remote.php/dav/principals/users/username/ (replace example.com and username).

  9. Saisir votre nom d’utilisateur et le mot de passe.

  10. Sélectionner Suivant.

Vous devriez maintenant trouver vos contacts dans le carnet d’adresses de votre iPhone.

Note

À partir de iOS 12, un cryptage SSL est nécessaire. Par conséquent, ne désactivez pas le SSL (Pour cette raison, un certificat est requis pour votre domaine, comme https://letsencrypt.org/ ).

Si cela ne fonctionne toujours pas, regardez dans Dépannage des contacts et calendriers ou Dépannage de la découverte de service.
