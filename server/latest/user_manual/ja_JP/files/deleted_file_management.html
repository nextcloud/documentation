削除したファイルの管理

Nextcloud でファイルを削除しても、すぐに完全削除されるわけではなく、ゴミ箱に移動されるだけです。手動で削除するか「削除したファイル」アプリが新しいファイルのための空きを確保する目的で削除するまでは、完全には削除されません。

Find your deleted files by clicking the Deleted files button on the Files page of the Nextcloud web interface. You can restore or permanently delete files from there.

クォータ

削除したファイルはストレージクォータの対象になりません。クォータの対象となるのは個人用ファイルのみで、自分へ共有されたファイルは対象になりません。（クォータの詳細は Storage quota を参照してください。）

共有ファイルが削除された場合の動作

共有ファイルを削除する場合は少し複雑になります。次のシナリオを例に説明します:

  1. ユーザ1 がユーザ2 とユーザ3 にフォルダ「test」を共有しています

  2. ユーザ2（受信者）が「test」の中のファイル/フォルダ「sub」を削除します

  3. フォルダ「sub」は、ユーザ1（所有者）とユーザ2（受信者）の両方のゴミ箱に移動されます

  4. ただしユーザ3 のゴミ箱には「sub」のコピーは保存されません

ユーザ1 が「sub」を削除すると、「sub」はユーザ1 のゴミ箱に移動します。ユーザ2 とユーザ3 からは削除されますが、両者のゴミ箱には入れられません。

ファイルを共有すると、他のユーザは通常のコンピュータのファイルと同様に、コピー、名前変更、移動、さらに別の人への共有が可能です。Nextcloud には、これを魔法のように防ぐ力はありません。

「削除したファイル」アプリによるストレージ容量の管理

ユーザがストレージクォータを超えないように、「削除したファイル」アプリは、現在利用可能な空き容量の最大 50% を削除したファイルに割り当てます。削除したファイルがこの上限を超えた場合、Nextcloud は最も古いファイル（削除された時刻が最も古いファイル）から削除し、再び上限内に収まるまで削除を続けます。

Nextcloud は、新しいファイルが「削除したファイル」に追加されるたびに、削除したファイルの経過日数を確認します。デフォルトでは、削除したファイルは 30 日間ゴミ箱に保持されます。Nextcloud サーバ管理者は、config.php ファイルで trashbin_retention_obligation の値を設定することで、この期間を調整できます。trashbin_retention_obligation の値より古いファイルは完全に削除されます。さらに Nextcloud は、新しいファイルが追加されるたびに最大利用可能容量を計算します。削除したファイルが新しい上限を超えた場合、期限が最も早く切れるゴミ箱内のファイルから完全に削除し、再び容量上限内に収まるまで削除を続けます。

注釈

管理者が、ストレージ容量の管理よりも優先されるように、ゴミ箱の保持期間を設定している場合があります。詳細は 管理者ドキュメント を参照してください。
