ການຈັດການໄຟລ໌ທີ່ຖືກລຶບ

ເມື່ອທ່ານລຶບໄຟລ໌ໃນ Nextcloud, ມັນຈະບໍ່ຖືກລຶບຖິ້ມຢ່າງຖາວອນໃນທັນທີ, ແຕ່ຈະຖືກຍ້າຍໄປໄວ້ໃນຖັງຂີ້ເຫຍື້ອເທົ່ານັ້ນ. ມັນຈະບໍ່ຖືກລຶບຖິ້ມຢ່າງຖາວອນ ຈົນກວ່າທ່ານຈະລຶບມັນດ້ວຍຕົນເອງ, ຫຼື ເມື່ອແອັບ "ໄຟລ໌ທີ່ຖືກລຶບ" Deleted Files ລຶບມັນອອກເພື່ອເພີ່ມພື້ນທີ່ຫວ່າງໃຫ້ກັບໄຟລ໌ໃໝ່.

ຄົ້ນຫາໄຟລ໌ທີ່ຖືກລຶບຂອງທ່ານໂດຍການຄລິກໃສ່ປຸ່ມ ໄຟລ໌ທີ່ຖືກລຶບ Deleted files ໃນໜ້າ "ໄຟລ໌" ຂອງເວັບອິນເຕີເຟດ Nextcloud. ທ່ານຈະມີຕົວເລືອກເພື່ອ ກູ້ຄືນ ຫຼື ລຶບໄຟລ໌ຖິ້ມຢ່າງຖາວອນ.

ໂຄຕ້າ

ໄຟລ໌ທີ່ຖືກລຶບຈະບໍ່ຖືກນັບລວມເຂົ້າໃນໂຄຕ້າການຈັດເກັບຂໍ້ມູນຂອງທ່ານ. ມີພຽງແຕ່ໄຟລ໌ສ່ວນຕົວຂອງທ່ານເທົ່ານັ້ນທີ່ຈະຖືກນັບເຂົ້າໃນໂຄຕ້າ, ບໍ່ແມ່ນໄຟລ໌ທີ່ມີຄົນແບ່ງປັນໃຫ້ທ່ານ. (ເບິ່ງ :doc:quota ເພື່ອຮຽນຮູ້ເພີ່ມເຕີມກ່ຽວກັບໂຄຕ້າ.)

ຈະເກີດຫຍັງຂຶ້ນເມື່ອໄຟລ໌ທີ່ຖືກແບ່ງປັນຖືກລຶບ

ການລຶບໄຟລ໌ອາດຈະມີຄວາມຊັບຊ້ອນເລັກນ້ອຍເມື່ອເປັນໄຟລ໌ທີ່ຖືກແບ່ງປັນ, ດັ່ງທີ່ສະຖານະການນີ້ສະແດງໃຫ້ເຫັນ:

  1. User1 ແບ່ງປັນໂຟນເດີ "test" ໃຫ້ກັບ User2 ແລະ User3

  2. User2 (ຜູ້ຮັບ) ລຶບໄຟລ໌/ໂຟນເດີ "sub" ທີ່ຢູ່ທາງໃນ "test"

  3. ໂຟນເດີ "sub" ຈະຖືກຍ້າຍໄປຢູ່ໃນຖັງຂີ້ເຫຍື້ອຂອງທັງ User1 (ເຈົ້າຂອງ) ແລະ User2 (ຜູ້ຮັບ)

  4. ແຕ່ User3 ຈະບໍ່ມີສຳເນົາຂອງ "sub" ໃນຖັງຂີ້ເຫຍື້ອຂອງພວກເຂົາ

ເມື່ອ User1 ລຶບ "sub" ມັນຈະຖືກຍ້າຍໄປໄວ້ໃນຖັງຂີ້ເຫຍື້ອຂອງ User1. ມັນຈະຖືກລຶບອອກຈາກ User2 ແລະ User3, ແຕ່ຈະບໍ່ຖືກນຳໄປໄວ້ໃນຖັງຂີ້ເຫຍື້ອຂອງພວກເຂົາ.

ເມື່ອທ່ານແບ່ງປັນໄຟລ໌, ຜູ້ໃຊ້ອື່ນອາດຈະຄັດລອກ, ປ່ຽນຊື່, ຍ້າຍ ແລະ ແບ່ງປັນໃຫ້ກັບຄົນອື່ນ, ຄືກັນກັບທີ່ພວກເຂົາສາມາດເຮັດໄດ້ກັບໄຟລ໌ຄອມພິວເຕີທົ່ວໄປ; Nextcloud ບໍ່ມີພະລັງວິເສດທີ່ຈະປ້ອງກັນເລື່ອງນີ້ໄດ້.

ວິທີທີ່ແອັບໄຟລ໌ທີ່ຖືກລຶບຈັດການພື້ນທີ່ຈັດເກັບຂໍ້ມູນ

ເພື່ອຮັບປະກັນວ່າຜູ້ໃຊ້ຈະບໍ່ໃຊ້ພື້ນທີ່ເກີນໂຄຕ້າການຈັດເກັບຂໍ້ມູນ, ແອັບ ໄຟລ໌ທີ່ຖືກລຶບ ຈະຈັດສັນພື້ນທີ່ສູງສຸດ 50% ຂອງພື້ນທີ່ຫວ່າງທີ່ມີຢູ່ໃນປັດຈຸບັນໃຫ້ກັບໄຟລ໌ທີ່ຖືກລຶບ. ຖ້າໄຟລ໌ທີ່ຖືກລຶບຂອງທ່ານເກີນຂີດຈຳກັດນີ້, Nextcloud ຈະລຶບໄຟລ໌ທີ່ເກົ່າທີ່ສຸດ (ໄຟລ໌ທີ່ມີວັນເວລາທີ່ຖືກລຶບດົນທີ່ສຸດ) ຈົນກວ່າຈະກັບມາຢູ່ໃນຂີດຈຳກັດການໃຊ້ຄວາມຈຳອີກຄັ້ງ.

Nextcloud ຈະກວດສອບອາຍຸຂອງໄຟລ໌ທີ່ຖືກລຶບທຸກຄັ້ງທີ່ມີການເພີ່ມໄຟລ໌ໃໝ່ເຂົ້າໄປໃນລາຍການໄຟລ໌ທີ່ຖືກລຶບ. ໂດຍຄ່າເລີ່ມຕົ້ນ, ໄຟລ໌ທີ່ຖືກລຶບຈະຢູ່ໃນຖັງຂີ້ເຫຍື້ອເປັນເວລາ 30 ວັນ. ຜູ້ດູແລລະບົບເຊີບເວີ Nextcloud ສາມາດປັບຄ່ານີ້ໄດ້ໃນໄຟລ໌ config.php ໂດຍການຕັ້ງຄ່າ trashbin_retention_obligation. ໄຟລ໌ທີ່ມີອາຍຸເກີນກວ່າຄ່າ trashbin_retention_obligation ຈະຖືກລຶບຖິ້ມຢ່າງຖາວອນ. ນອກຈາກນັ້ນ, Nextcloud ຍັງຄຳນວນພື້ນທີ່ຫວ່າງສູງສຸດທຸກຄັ້ງທີ່ມີການເພີ່ມໄຟລ໌ໃໝ່. ຖ້າໄຟລ໌ທີ່ຖືກລຶບເກີນພື້ນທີ່ສູງສຸດທີ່ອະນຸຍາດໃໝ່, Nextcloud ຈະລຶບໄຟລ໌ໃນຖັງຂີ້ເຫຍື້ອທີ່ໃກ້ຈະໝົດອາຍຸທີ່ສຸດຖາວອນ ຈົນກວ່າຈະຮອດຂີດຈຳກັດພື້ນທີ່ອີກຄັ້ງ.

Note

ຜູ້ດູແລລະບົບຂອງທ່ານອາດຈະຕັ້ງຄ່າໄລຍະເວລາການເກັບຮັກສາໃນຖັງຂີ້ເຫຍື້ອ ເພື່ອແທນທີ່ການຈັດການພື້ນທີ່ຈັດເກັບຂໍ້ມູນ. ເບິ່ງ ເອກະສານສຳລັບຜູ້ດູແລລະບົບ <https://docs.nextcloud.com/server/latest/admin_manual/configuration_server/config_sample_php_parameters.html#deleted-items-trash-bin>_ ສຳລັບລາຍລະອຽດເພີ່ມເຕີມ.
