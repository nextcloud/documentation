Komma åt Nextcloud-filer med WebDAV

Nextcloud fully supports the WebDAV protocol, and you can connect to and synchronize with Nextcloud Files over WebDAV. This chapter explains how to connect Linux, macOS, Windows, and mobile devices to your Nextcloud server.

WebDAV står för Distributed Authoring and Versioning. Det är ett tillägg till HTTP som gör det enkelt att skapa, läsa och redigera filer på fjärranslutna webbservrar. Med en WebDAV-klient kan du komma åt dina Nextcloud-filer, inklusive delningar, i Linux, macOS och Windows på ungefär samma sätt som en fjärransluten nätverksdelning och hålla dem synkroniserade.

Before you configure WebDAV, review the recommended way to connect client devices to Nextcloud.

Officiella Nextcloud-klienter för datorer och mobila enheter

Det rekommenderade sättet att synkronisera en dator med en Nextcloud-server är att använda de officiella synkroniseringsklienterna för Nextcloud. Du kan konfigurera klienten så att den sparar filer i valfri lokal mapp och välja vilka mappar på Nextcloud-servern som ska synkroniseras. Klienten visar aktuell anslutningsstatus och loggar all aktivitet. Därmed vet du alltid vilka fjärrfiler som har hämtats till datorn och kan kontrollera att filer som skapats eller uppdaterats lokalt synkroniseras korrekt med servern.

Det rekommenderade sättet att synkronisera Android- och Apple iOS-enheter är att använda de officiella mobilapparna för Nextcloud.

Anslut de officiella Nextcloud-apparna till en Nextcloud-server med samma URL som du använder för att öppna Nextcloud i webbläsaren, till exempel:

https://cloud.example.com

Om Nextcloud är installerat i en undermapp som heter ”nextcloud”:

https://example.com/nextcloud

WebDAV-klienter från tredje part

Du kan också ansluta datorn till Nextcloud-servern med valfri tredjepartsklient som stöder WebDAV-protokollet, inklusive klienten som kan vara inbyggd i operativsystemet.

Du kan även använda tredjepartsappar med WebDAV-stöd för att ansluta en mobil enhet till Nextcloud.

Tänk på att tredjepartsklienter kanske inte är optimerade för Nextcloud eller saknar funktioner som är viktiga för ditt användningsområde.

Mobilklienter som medlemmar i Nextcloud-gemenskapen har uppgett att de använder är bland annat:

  • FolderSync (Android)

  • WebDAV Navigator (iPhone)

URL:en som används när tredjepartsappar konfigureras för anslutning till Nextcloud är något längre än den som används för officiella klienter:

https://cloud.example.com/remote.php/dav/files/USERNAME/

Om Nextcloud är installerat i en undermapp som heter ”nextcloud”:

https://example.com/nextcloud/remote.php/dav/files/USERNAME/

Observera

When using a third-party WebDAV client (including your operating system’s built-in client), you should use an application password for login rather than your regular password. In addition to improved security, this increases performance significantly. To configure an application password, log in to the Nextcloud Web interface, click your avatar, and choose Personal settings. Then choose Security in the left sidebar and scroll to the very bottom. There you can create an app password (which can also be revoked in the future without changing your main user password).

Observera

I följande exempel ska du ersätta example.com/nextcloud med URL:en till din Nextcloud-server. Utelämna mappsökvägen om installationen finns i domänens rot. Ersätt USERNAME med användarnamnet för den användare som ansluter.

You can find the WebDAV URL in Files settings under WebDAV in your Nextcloud account.

Komma åt filer i Linux

Du kan komma åt filer i Linux-operativsystem med följande metoder.

Filhanteraren Nautilus

When you configure your Nextcloud account in the GNOME Control Center, your files will automatically be mounted by Nautilus as a WebDAV share, unless you deselect file access.

Du kan också montera Nextcloud-filerna manuellt. Använd protokollet davs:// för att ansluta filhanteraren Nautilus till Nextcloud-delningen:

davs://example.com/nextcloud/remote.php/dav/files/USERNAME/

Observera

Om serveranslutningen inte är skyddad med HTTPS använder du dav:// i stället för davs://:

Observera

Samma metod fungerar för andra filhanterare som använder GVFS, till exempel Caja i MATE och Nemo i Cinnamon.

Accessing files with KDE and Dolphin

  1. Gå till Systeminställningar → Nätverk → Nätkonton

  2. Klicka på ”Lägg till konto …”

  3. Klicka på Nextcloud

  4. Ange serveradressen

  5. Följ anvisningarna på skärmen för att logga in

  6. När du har loggat in ska du aktivera ”Lagring” i avsnittet ”Använd det här kontot för”

  7. Nu kan du komma åt filerna under ”Nätverk” i Dolphins sidofält

  8. (Optional) To add this location as a shortcut in the sidebar, right-click ”Nextcloud Storage”, then click ”Add to Places”

  9. (Optional) To customize the shortcut, right-click it in the sidebar, then click ”Edit…” and customize the icon and label

Skapa WebDAV-monteringar via Linux-kommandoraden

Du kan skapa WebDAV-monteringar från Linux-kommandoraden. Det är användbart om du vill komma åt Nextcloud på samma sätt som ett annat fjärrmonterat filsystem. Följande exempel visar hur du skapar en personlig montering som monteras automatiskt varje gång du loggar in på Linux-datorn.

  1. Installera WebDAV-filsystemdrivrutinen davfs2, som gör att WebDAV-delningar kan monteras på samma sätt som andra fjärrfilsystem. Installera den i Debian eller Ubuntu med följande kommando:

    apt-get install davfs2

  2. Installera den i CentOS, Fedora eller openSUSE med följande kommando:

    yum install davfs2

  3. Lägg till dig själv i gruppen davfs2:

    usermod -aG davfs2 <username>

Observera

If the davfs2 group doesn’t exist after installing the package, you may need to create it yourself and, possibly, adjust the davfs config file to use the group after you’ve created it.

  1. Skapa sedan mappen nextcloud i hemkatalogen som monteringspunkt och mappen .davfs2/ för den personliga konfigurationsfilen:

    mkdir ~/nextcloud
mkdir ~/.davfs2

  2. Kopiera /etc/davfs2/secrets till ~/.davfs2:

    cp  /etc/davfs2/secrets ~/.davfs2/secrets

  3. Ange dig själv som ägare och ge endast ägaren läs- och skrivbehörighet:

    chown <linux_username>:<linux_username> ~/.davfs2/secrets
chmod 600 ~/.davfs2/secrets

  4. Lägg till dina inloggningsuppgifter för Nextcloud sist i filen secrets. Använd URL:en till Nextcloud-servern samt ditt Nextcloud-användarnamn och lösenord:

    https://example.com/nextcloud/remote.php/dav/files/USERNAME/ <username> <password>
or
$PathToMountPoint $USERNAME $PASSWORD
for example
/home/user/nextcloud john 1234

  5. Lägg till monteringsinformationen i /etc/fstab:

    https://example.com/nextcloud/remote.php/dav/files/USERNAME/ /home/<linux_username>/nextcloud davfs user,rw,auto 0 0

  6. Kontrollera sedan att monteringen och autentiseringen fungerar genom att köra följande kommando. Om allt är korrekt konfigurerat behöver du inte rotbehörighet:

    mount ~/nextcloud

  7. Du bör också kunna avmontera den:

    umount ~/nextcloud

Varje gång du loggar in på Linux-systemet bör Nextcloud-delningen nu monteras automatiskt via WebDAV i mappen ~/nextcloud. Vill du montera den manuellt ändrar du auto till noauto i /etc/fstab.

Kända problem

Problem

Resursen är tillfälligt otillgänglig

Lösning

Om du får problem när du skapar en fil i mappen redigerar du /etc/davfs2/davfs2.conf och lägger till:

use_locks 0

Problem

Certifikatvarningar

Lösning

Om du använder ett självsignerat certifikat visas en varning. Konfigurera davfs2 så att certifikatet identifieras genom att kopiera mycertificate.pem till /etc/davfs2/certs/. Redigera sedan /etc/davfs2/davfs2.conf och ta bort kommentartecknet från raden servercert. Lägg därefter till sökvägen till certifikatet enligt följande exempel:

servercert /etc/davfs2/certs/mycertificate.pem

Komma åt filer i macOS

Så här kommer du åt filer via Finder i macOS:

  1. Välj Gå > Anslut till server … i Finders övre menyrad:

  2. När fönstret Anslut till server … öppnas anger du Nextcloud-serverns WebDAV-adress i fältet Serveradress:, det vill säga:

    https://cloud.YOURDOMAIN.com/remote.php/dav/files/USERNAME/

  3. Klicka på Anslut. WebDAV-servern bör visas på skrivbordet som en delad diskenhet.

Komma åt filer i Microsoft Windows

Om du använder Windows inbyggda WebDAV-funktion kan du ansluta Nextcloud som en ny enhet i Utforskaren. När enheten har anslutits kan du bläddra bland filer på Nextcloud-servern på samma sätt som på en ansluten nätverksenhet.

Funktionen kräver en nätverksanslutning. Vill du lagra filerna offline använder du skrivbordsklienten för att synkronisera alla filer i Nextcloud till en eller flera mappar på den lokala hårddisken.

Observera

In Windows 10, Basic Authentication is allowed by default when HTTPS is enabled before mapping your drive.

I äldre Windows-versioner måste du tillåta grundläggande autentisering i Windows-registret:

  • Starta regedit och gå till HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\WebClient\Parameters.

  • Skapa eller redigera DWORD-värdet BasicAuthLevel (Windows Vista, 7 och 8) eller UseBasicAuth (Windows XP och Windows Server 2003) och ange värdedata 1 för SSL-anslutningar. Värdet 0 innebär att grundläggande autentisering är inaktiverad, medan 2 tillåter både SSL- och icke-SSL-anslutningar, vilket inte rekommenderas.

  • Avsluta sedan Registereditorn och starta om datorn.

Ansluta enheter via kommandoraden

Följande exempel visar hur du ansluter en enhet via kommandoraden. Så här ansluter du enheten:

  1. Öppna Kommandotolken i Windows.

  2. Ange följande rad i Kommandotolken för att ansluta till enhet Z på datorn:

    net use Z: https://<drive_path>/remote.php/dav/files/USERNAME/ /user:youruser yourpassword

    där <drive_path> är URL:en till Nextcloud-servern. Exempel:

    net use Z: https://example.com/nextcloud/remote.php/dav/files/USERNAME/ /user:youruser yourpassword

    Filerna i ditt Nextcloud-konto ansluts till enhetsbeteckningen Z.

Fel

Om felet System error 67 has occurred. The network name cannot be found. visas eller anslutningen bryts ofta öppnar du appen Tjänster och kontrollerar att tjänsten WebClient körs och startas automatiskt när systemet startar.

Observera

Även om det inte rekommenderas kan du montera Nextcloud-servern via HTTP, vilket lämnar anslutningen okrypterad.

Om du tänker använda HTTP-anslutningar på enheter på offentliga platser rekommenderar vi starkt en VPN-tunnel som ger nödvändigt skydd.

En alternativ kommandosyntax är:

net use Z: \\example.com@ssl\nextcloud\remote.php\dav /user:youruser
yourpassword

Ansluta enheter med Utforskaren

Så här ansluter du en enhet med Utforskaren i Microsoft Windows:

  1. Öppna Utforskaren på Windows-datorn.

  2. Högerklicka på Dator och välj Anslut nätverksenhet … i listrutan.

  3. Välj den lokala enhetsbeteckning som du vill ansluta Nextcloud till.

  4. Ange adressen till Nextcloud-instansen följd av /remote.php/dav/files/USERNAME/.

    Exempel:

    https://example.com/nextcloud/remote.php/dav/files/USERNAME/

    Observera

    För SSL-skyddade servrar markerar du Återanslut vid inloggning så att anslutningen finns kvar efter omstarter. Vill du ansluta till Nextcloud-servern som en annan användare markerar du Anslut med andra autentiseringsuppgifter.

  5. Klicka på knappen Slutför.

Utforskaren ansluter nätverksenheten och gör Nextcloud-instansen tillgänglig.

Komma åt filer med Cyberduck

Cyberduck är ett program med öppen källkod för att bläddra i och överföra filer via FTP, SFTP, WebDAV, OpenStack Swift och Amazon S3 i macOS och Windows.

Observera

I det här exemplet används Cyberduck version 4.2.1.

Så här använder du Cyberduck:

  1. Ange servern utan någon inledande protokollangivelse.

    Exempel: example.com

  2. Ange rätt port.

    Vilken port du väljer beror på om Nextcloud-servern stöder SSL. I Cyberduck måste du välja en annan anslutningstyp om du tänker använda SSL.

    Exempel:

    • 80 för okrypterad WebDAV

    • 443 för säker WebDAV (HTTPS/SSL)

  3. Använd listrutan ”Fler alternativ” och lägg till resten av WebDAV-URL:en i fältet ”Sökväg”.

    Exempel: remote.php/dav/files/USERNAME/

Cyberduck ger nu åtkomst till filerna på Nextcloud-servern.

Komma åt offentliga delningar via WebDAV

Nextcloud gör det möjligt att komma åt offentliga delningar anonymt via WebDAV.

Öppna följande för att komma åt den offentliga delningen:

https://example.com/nextcloud/public.php/dav/files/USERNAME

Använd delningstoken som användarnamn och det valfria delningslösenordet som lösenord i en WebDAV-klient. För delningslänken https://example.com/s/kFy9Lek5sm928xP är exempelvis kFy9Lek5sm928xP användarnamnet.

Observera

Inställningar → Administration → Delning → Tillåt användare på den här servern att skicka delningar till andra servrar. Alternativet tillåter också WebDAV-åtkomst till offentliga delningar och måste vara aktiverat för att funktionen ska fungera, utom när cURL används (se nedan).

Kända problem

Problem

WebDAV access is very slow or times out.

Lösning

When using a third-party WebDAV client (including your operating system’s built-in client), you should authenticate with an application password rather than your regular account password. Using a regular password carries a significant performance penalty because Nextcloud must perform additional work to verify it on every request. Using an application password removes this overhead and can improve transfer speeds substantially.

To create an application password, log in to the Nextcloud web interface, click your avatar, choose Personal settings, then open the Security section in the left sidebar and scroll to the bottom. Create an app password there and use it in your WebDAV client instead of your main password.

Problem

Windows ansluter inte via HTTPS.

Lösning 1

Windows WebDAV-klient kanske inte stöder Server Name Indication (SNI) för krypterade anslutningar. Om ett fel uppstår när du monterar en SSL-krypterad Nextcloud-instans kontaktar du leverantören och frågar om servern kan få en särskild IP-adress för SSL.

Lösning 2

Windows WebDAV-klient kanske inte stöder anslutningar med TLSv1.1 och TLSv1.2. Om serverkonfigurationen endast tillåter TLSv1.1 och senare kan anslutningen till servern misslyckas. Mer information finns i dokumentationen för WinHTTP.

Problem

Följande felmeddelande visas: Error 0x800700DF: The file size exceeds the limit allowed and cannot be saved.

Lösning

Windows begränsar den största tillåtna storleken för filer som överförs till eller från en WebDAV-delning. Du kan höja värdet FileSizeLimitInBytes i HKEY_LOCAL_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\WebClient\\Parameters genom att klicka på Ändra.

Höj gränsen till det högsta värdet 4 GB genom att välja Decimal, ange 4294967295 och starta om Windows eller tjänsten WebClient.

Problem

Adding a WebDAV drive on Windows with the steps above does not display the correct available Nextcloud space and instead shows the size and free space of the C: drive.

Svar

Unfortunately, this is a limitation of WebDAV itself, because it does not provide a way for the client to retrieve available free space from the server. Windows automatically falls back to the size and free space of the C: drive. There is no direct solution for this limitation.

Problem

Det går inte att komma åt filerna från Microsoft Office via WebDAV.

Lösning

Kända problem och lösningar dokumenteras i artikeln KB2123563.

Problem

Det går inte att ansluta Nextcloud som en WebDAV-enhet i Windows med ett självsignerat certifikat.

Lösning

  1. Access your Nextcloud instance in your preferred Web browser.

  2. Fortsätt tills certifikatfelet visas på webbläsarens statusrad.

  3. Visa certifikatet och välj sedan ”Kopiera till fil” på fliken Detaljer.

  4. Spara filen på skrivbordet med ett valfritt namn, till exempel myNextcloud.pem.

  5. Öppna Start-menyn > Kör, skriv MMC och klicka på ”OK” för att öppna Microsoft Management Console.

  6. Gå till Arkiv > Lägg till/ta bort snapin-modul.

  7. Välj Certifikat, klicka på ”Lägg till”, välj ”Mitt användarkonto”, klicka på ”Slutför” och slutligen på ”OK”.

  8. Gå till Betrodda rotcertifikatutfärdare och sedan Certifikat.

  9. Högerklicka på Certifikat och välj Alla aktiviteter > Importera.

  10. Välj det sparade certifikatet på skrivbordet.

  11. Välj Placera alla certifikat i följande arkiv och klicka på Bläddra.

  12. Markera rutan Visa fysiska arkiv, expandera Betrodda rotcertifikatutfärdare, välj Lokal dator, klicka på ”OK” och slutför importen.

  13. Kontrollera i listan att certifikatet visas. Du kan behöva uppdatera vyn innan det syns.

  14. Avsluta MMC.

För Firefox-användare:

  1. Starta webbläsaren och gå till Programmeny > Historik > Rensa ut tidigare historik …

  2. Välj ”All historik” i listrutan ”Tidsintervall att ta bort”

  3. Markera kryssrutan ”Aktiva inloggningar”

  4. Klicka på knappen ”Ta bort”

  5. Stäng webbläsaren, öppna den igen och testa.

För användare av Chromium-baserade webbläsare (Chrome, Chromium och Microsoft Edge):

  1. Öppna Kontrollpanelen i Windows och gå till Internetalternativ

  2. Klicka på knappen Rensa SSL-tillstånd på fliken Innehåll.

  3. Stäng webbläsaren, öppna den igen och testa.

Komma åt filer med cURL

Eftersom WebDAV är ett tillägg till HTTP kan cURL användas för att skripta filåtgärder.

Observera

Inställningar → Administration → Delning → Tillåt användare på den här servern att skicka delningar till andra servrar. Om alternativet är inaktiverat måste flaggan --header "X-Requested-With: XMLHttpRequest" skickas till cURL.

Så här skapar du en mapp med dagens datum som namn:

$ curl -u user:pass -X MKCOL "https://example.com/nextcloud/remote.php/dav/files/USERNAME/$(date '+%d-%b-%Y')"

Så här laddar du upp filen error.log till mappen:

$ curl -u user:pass -T error.log "https://example.com/nextcloud/remote.php/dav/files/USERNAME/$(date '+%d-%b-%Y')/error.log"

Så här flyttar du en fil:

$ curl -u user:pass -X MOVE --header 'Destination: https://example.com/nextcloud/remote.php/dav/files/USERNAME/target.jpg' https://example.com/nextcloud/remote.php/dav/files/USERNAME/source.jpg

Komma åt filer med WinSCP

WinSCP is a free, open-source SFTP, FTP, WebDAV, S3, and SCP client for Windows. Its main function is file transfer between a local and a remote computer. WinSCP also offers scripting and basic file management functionality.

Du kan hämta den portabla versionen av WinSCP och köra den i Linux via Wine.

To run WinSCP on Linux, install Wine through your distribution’s package manager, then run: wine WinSCP.exe.

Så här ansluter du till Nextcloud:

  • Starta WinSCP

  • Klicka på ”Session” i menyn

  • Klicka på menyalternativet ”New Session”

  • Ange WebDAV i listrutan ”File protocol”

  • Ange TLS/SSL Implicit encryption i listrutan ”Encryption”

  • Fyll i värdnamnsfältet: example.com

  • Fyll i användarnamnsfältet: NEXTCLOUDUSERNAME

  • Fyll i lösenordsfältet: NEXTCLOUDPASSWORD

  • Klicka på knappen ”Advanced …”

  • Gå till ”Environment” och sedan ”Directories” till vänster

  • Fyll i fältet ”Remote directory” med följande: /nextcloud/remote.php/dav/files/NEXTCLOUDUSERNAME/

  • Klicka på knappen ”OK”

  • Klicka på knappen ”Save”

  • Välj önskade alternativ och klicka på knappen ”OK”

  • Klicka på knappen ”Login” för att ansluta till Nextcloud

Observera

If you use TOTP, use an app password. At the time of writing (2022-11-07), WinSCP does not support TOTP with Nextcloud.
